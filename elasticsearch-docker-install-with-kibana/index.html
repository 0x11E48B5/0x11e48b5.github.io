What is Elasticsearch?

Elasticsearch is a distributed search and analytics engine built on Apache Lucene. It is designed for scalability, speed, and full-text search, making it widely used for log analysis, real-time monitoring, business intelligence, and security analytics. Here is the key features of elasticsearch.

  • Full-Text Search – Powerful text search capabilities with relevance ranking.
  • Scalability – Can handle massive amounts of data by distributing workloads across multiple nodes.
  • Near Real-Time Search – Indexes and retrieves data almost instantly.
  • RESTful API – Allows easy integration with various applications.
  • Schema-Free JSON Storage – Accepts semi-structured JSON data without requiring predefined schemas.
  • Aggregation Framework – Enables advanced data analysis (e.g., trends, statistics).
  • Elasticsearch Query DSL – A flexible and powerful query language for retrieving data.

In this tutorial, we will deploy single-node elasticsearch and kibana in ubuntu. To create a multi-node cluster with kibana, you should use docker compose instead. See multi-node cluster using docker compose

Let’s try to explain all arguments in that command.

  • Firstly the -d parameter is (detached mode) that runs the container in the background.
  • --name es01 parameter assigns a name (es01) to the container for easy reference.
  • --net elastic specifies a custom Docker network named elastic for communication with other services (e.g., Kibana).
  • --ip 172.18.0.2 assigns a static IP (172.18.0.2) within the elastic network. We should assign this ip adress for this deployment type because while installing kibana you will see, the kibana yaml looks this ip address for elasticsearch in default. If kibana don’t find elasticsearch in that ip address within the network, it crashes.
  • -p 9200:9200 maps port 9200 of the container (Elasticsearch’s default HTTP port) to port 9200 on the host machine.
  • -m 16GB limits the container’s memory usage to 16GB, preventing excessive memory consumption.
  • -v /mnt/elasticsearch/data:/usr/share/elasticsearch/data mounts a persistent volume:
    • Host directory: /mnt/elasticsearch/data
    • Container directory: /usr/share/elasticsearch/data
      Ensures Elasticsearch data is retained even if the container is removed.
  • -e "bootstrap.memory_lock=true" enables memory locking to prevent Elasticsearch from swapping memory to disk.
  • --ulimit memlock=-1:-1 grants unlimited memory lock permissions to support bootstrap.memory_lock=true.
  • --restart=always ensures the container automatically restarts if it crashes or if the host machine reboots.

After runnign the command, you can check the logs using docker logs -f es01. When installation process was fineshed, there will be outputs on your shell taht shows enrollment-token, elastic user password we must note that values for using in the future.
